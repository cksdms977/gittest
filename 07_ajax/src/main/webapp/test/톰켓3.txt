bin : 톰켓을 실행하는 파일 -> 
conf : 톰겟에 실행할때 필요한 환경설정파일 -> xml파일 (서버동작환경설정 -> server.xml) (서버의 애플리케이션 설정-> web xml)
lib : jar필요한 외부 라이브러리 보관
logs : 톰켓이 필요한 log(기록),에러에 대한 파일을 저장
temp, work : 톰켓이 실행될때 사용하는 공간 
webapps : 내가 만든 web서비스를 베포하는 폴더(소스코드를 주는게 아닌 패키징->war파일을 이용함)

web.xml : 베포서술자 -> webservice구동 -> 환경설정파일
(기본공통적인 설정파일 1개, 어플리케이션 만들때마다 생성되는 파일 = 이게 먼저실행되고 기본설정파일에 있으면 실행 없으면 실행하지 않음 )

context-param : 공용데이터 key:value형식으로 저장(서버상 전역변수(데이터))
servlet : 요청한 url값을 맵핑해서 저장 이름과 클래스명 모두 작성
